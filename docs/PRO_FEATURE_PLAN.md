# Pro 版本功能規劃（MVP-5）

> roadmap 項目：**MVP-5：用戶體驗與增長 / Pro 版本功能規劃（如預算、進階報表）**

## 1) 目標與原則

### 目標
1. 提升留存：讓重度使用者在第 2～4 週持續打開 App。
2. 建立付費價值：以「可衡量的財務管理成果」驅動升級，而不是單純功能堆疊。
3. 降低導入成本：維持既有免費版核心流程不受干擾。

### 設計原則
- **先解痛點再賣功能**：Pro 功能要直接解決「控制支出」與「看懂花費趨勢」。
- **可漸進開啟**：先做讀取型能力（報表）再做行動型能力（預算與提醒自動化）。
- **跨平台一致**：iOS / Android 同步提供核心 Pro 能力，文案與限制一致。

---

## 2) Pro 功能範圍（第一波）

## A. 預算系統（Budgeting）

### 使用者價值
- 讓使用者可為「月份 + 分類」設定預算，追蹤剩餘可用額度。
- 支援每月固定預算與滾動結轉（進階選項）。

### 功能細節
1. **預算建立與管理**
   - 維度：`month`、`categoryId`、`amount`、`carryOverMode`。
   - 支援快速複製上月預算。
2. **預算消耗進度**
   - 顯示 `已花費 / 預算 / 剩餘`。
   - 進度條顏色：正常（<80%）、警示（80~100%）、超支（>100%）。
3. **提醒策略**
   - 達 80% 時提醒（可關閉）。
   - 超支提醒（預設開啟）。
4. **免費版限制（建議）**
   - 每月最多 2 個分類預算。
   - 不提供結轉模式。

## B. 進階報表（Advanced Reports）

### 使用者價值
- 幫使用者回答「錢花去哪裡」、「和上個月比有沒有改善」。

### 功能細節
1. **趨勢圖（3/6/12 月）**
   - 總支出趨勢、收入趨勢、淨流量趨勢。
2. **分類變化分析**
   - Top 增長分類、Top 下降分類（MoM）。
3. **週期性支出洞察**
   - 訂閱 + 分期 + 常見週期型支出合併檢視。
4. **可匯出報表（Pro）**
   - 匯出 PDF（含摘要 + 圖表）。
5. **免費版限制（建議）**
   - 僅看當月摘要；歷史比較與 PDF 匯出鎖定 Pro。

## 3) 定價與權限牆（Paywall）策略

## 定價建議（可 A/B）
- 月付：NT$90
- 年付：NT$790（約 27% 折扣）
- 7 天免費試用（年付方案）

## 觸發點
1. 使用者建立第 3 個分類預算時。
2. 開啟 3 個月以上趨勢圖時。
3. 點擊匯出 PDF 報表時。

## Paywall 文案方向
- 以成果導向：「避免超支」、「每月回顧更快」、「找到可省下的分類」。
- 避免純技術描述（例如「解鎖圖表元件」）。

---

## 4) 事件追蹤與 KPI

## 核心事件（analytics）
- `pro_paywall_viewed`
- `pro_trial_started`
- `pro_subscribed`
- `budget_created`
- `budget_alert_triggered`
- `advanced_report_viewed`
- `report_pdf_exported`

## KPI（首兩個版本）
1. Paywall → 訂閱轉換率 >= 3%
2. 試用 → 付費轉換率 >= 25%
3. 啟用預算後 30 日留存提升 >= 8%
4. 使用進階報表使用者的週活躍天數提升 >= 10%

## 5) 實作拆解（工程規劃）

## 資料模型（建議新增）
- `BudgetPlan`
  - `id`, `month`, `categoryId`, `amount`, `carryOverMode`, `createdAt`, `updatedAt`
- `BudgetAlertPreference`
  - `threshold80Enabled`, `overspentEnabled`
- `ProEntitlement`
  - `tier`, `expiresAt`, `isTrial`, `source`

## iOS / Android 共同需求
1. Repository 新增 Budget 與 Report 查詢接口。
2. Domain 增加「預算計算器」與「分類變動分析器」。
3. UI 新增 Pro 標記與鎖定樣式（同一套 icon/文案規則）。
4. 付款狀態快取與離線 fallback（避免網路短暫失敗導致已付費使用者被鎖）。

## 版本分期
- **v1（2 週）**：預算 CRUD + 80%/超支提醒 + 3 個月趨勢圖 + 基本 Paywall
- **v1.1（2 週）**：分類變動分析 + 年付試用優化 + 轉換文案 A/B
- **v1.2（2 週）**：PDF 匯出 + 結轉預算 + 更完整週期支出洞察

---

## 6) 風險與緩解

1. **Paywall 過早打斷流程**
   - 緩解：只在高意圖操作觸發，不在初次記帳時觸發。
2. **報表計算效能問題**
   - 緩解：月資料快取 + 背景預先計算摘要。
3. **定價不匹配市場**
   - 緩解：以年付試用 + 兩組價格點做 A/B 驗證。
4. **平台體驗不一致**
   - 緩解：共用規格文件 + 同步驗收 checklist。
